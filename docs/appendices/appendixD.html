16. 附录 D：常见问题

本文档解答 RMTDataPro 使用中的常见问题。

16.1. 📋 通用问题

16.1.1. Q1: 软件启动失败怎么办？

A: 检查以下可能原因：

  1. 缺少 Visual C++ 运行库

    • 下载并安装 Visual Studio 2022 Redistributable

  2. Qt 插件加载失败

    • 确认 Qt 环境变量正确设置

    • 使用 -Deploy 参数重新部署 Qt 依赖

  3. 显卡驱动过旧

    • 更新显卡驱动到最新版本

16.1.2. Q2: 工程文件无法打开？

A: 可能原因及解决方法：

原因

解决方法

从备份恢复

版本不兼容

更新软件到最新版本

路径包含中文/特殊字符

移动到纯ASCII路径

16.1.3. Q3: 处理速度很慢？

A: 优化建议：

  1. 增加窗口重叠率 - 减少处理窗口数

  2. 减小窗口长度 - 降低频率分辨率

  3. 关闭详细日志 - 减少IO开销

  4. 使用 SSD - 提升数据读写速度

16.2. 📂 数据问题

16.2.1. Q4: SBF 文件加载失败？

A: 检查以下内容：

  1. 文件格式: 确认是 SBF 格式（非 AGE/SEG-D）

  2. 文件完整性: 尝试重新下载或复制文件

  3. 版本兼容: 部分旧版 SBF 可能不兼容

16.2.2. Q5: 某些频段数据缺失？

A: 可能原因：

  • 该频段在采集时未启用

  • 采集设备不支持该频段

  • 数据损坏导致解析失败

16.2.3. Q6: 处理结果异常（曲线抖动）？

A: 数据质量问题：

  1. 信噪比低: 延长采集时间

  2. 人文干扰: 远离电力线、建筑物

  3. 设备问题: 检查校准状态

16.4. 📤 导出问题

16.4.1. Q9: EDI 导出格式不被识别？

A: 检查 EDI 文件格式：

  • 确保包含 >HEAD 和 >MT DATA 标记

  • 确保 >END 标记存在

  • 确认频率值为正数

16.4.2. Q10: 批量导出失败？

A: 可能原因：

  1. 输出目录不存在或无写入权限

    • 创建目录或更换输出路径

  2. 文件名冲突

    • 启用”覆盖”选项

  3. 磁盘空间不足

    • 清理磁盘空间

16.5. 🌐 语言问题

16.5.1. Q11: 界面语言无法切换？

A: 解决方法：

  1. 确认翻译文件存在：

    • RMTDataPro_zh_CN.qm

    • RMTDataPro_en_US.qm

  2. 重启软件使语言切换生效

  3. 检查系统区域设置
